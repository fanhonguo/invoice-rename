## 更新内容

### 1. 修复发票解析问题 ✅

**问题描述：**
原程序无法正确识别增值税发票中的销售方名称和发票号码。

**原因分析：**
- 原正则表达式假设"销售方"和"发票号码"字段相邻
- 实际发票中这两个字段位于不同位置：
  - 发票号码位于顶部右侧
  - 销售方名称位于"销售方信息"区域的"名称"字段

**解决方案：**
更新了 `config.py` 中的正则表达式模式，新的模式可以：
- 正确匹配"发票号码："后的19-21位数字
- 从"销 名称："字段中提取销售方名称
- 处理PDF文本提取时的空格和换行变化

**测试结果：**
```
✓ 丹纳 3781885021.pdf → 上海丹纳国际物流有限公司
✓ 加壹 7155568051_.pdf → 广州加壹国际货运代理有限公司
✓ 港赛斐 3207486946_4月.pdf → 上 海 港赛斐国际物流有限公司
```

## 技术细节

### 正则表达式模式
```python
PATTERNS = [
    # 模式1: 匹配PDF提取时常见的格式 "销 名称：公司名"
    r'发票号码[：:]\s*(\d{15,21}).*?销\s名称[：:]\s*([^\s]{3,50}?\s*公司|...)'
    # 模式2: 匹配更宽松的格式
    r'发票号码[：:]\s*(\d{15,21}).*?销\s名称[：:]\s*([^\r\n]{3,50}?)\s*信'
    # 模式3: 最宽松的模式
    r'发票号码[：:]\s*(\d{15,21}).*?销\s名称[：:]\s*([^\r\n]+?)(?:\s*$|\r|\n)'
]
```
